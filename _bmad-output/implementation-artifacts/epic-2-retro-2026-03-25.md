## Enseignements clés

1. **L'investissement architectural en amont transforme les features complexes en livraisons fluides** — le GainNode obligatoire dès 1.2 a rendu le crossfade naturel. Quand une feature "potentiellement cauchemardesque" paraît simple, c'est que les fondations étaient bonnes
2. **Le bon grain de découpage est la cohérence fonctionnelle, pas la taille** — une story dense mais autonome vaut mieux que du micro-découpage qui génère plus de suivi que de production
3. **Projet temps libre = optimiser pour la qualité et le plaisir, pas la vélocité** — pas d'enjeu de rentabilité, pas de pression sur les délais. On fait ça pour le fun
4. **Les specs techniques détaillées éliminent l'incertitude** — formules sin/cos, anti-patterns documentés, architecture cible dessinée. Zéro debug log, zéro va-et-vient
5. **Le filet de sécurité des tests fonctionne** — 227 tests existants ont validé la refacto majeure d'audioEngine.js sans régression
